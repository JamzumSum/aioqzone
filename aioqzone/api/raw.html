class aioqzone.api.raw.QzoneApi(client, loginman)

Just a wrapper for Qzone http interface. No type validating.

async emotion_delete(fid, abstime, appid, typeid, topicId, uin=None)

Delete a feed.

Parameters

  • fid (str) – feed id, named tid, feedkey, etc.

  • abstime (int) – feed create time. now in emotion_publish() result dict.

  • appid (int) – appid

  • typeid (int) – typeid

  • topicId (str) – topic id, got from html

  • uin (Optional[int]) – host uin, defaults to None, means current logined user.

Raises

  • httpx.HTTPStatusError – error http response code

  • aioqzone.exception.QzoneError – error qzone response code

  • qqqr.exception.UserBreak – qr login canceled

  • aioqzone.exception.LoginError – not logined

  • SystemExit – unexcpected error

Return type

Optional[Dict[str, Union[bool, int, str, Dict[Union[str, int], Union[bool, int, str, Dict[Union[str, int], ForwardRef], List[ForwardRef]]], List[Union[bool, int, str, Dict[Union[str, int], ForwardRef], List[ForwardRef]]]]]]

Returns

qzone response as is, usually nothing meaningful.

New in version 0.2.6.
