Hooks

Define hooks that can trigger user actions.

class aioqzone.interface.hook.Emittable

An object has some event to trigger.

clear(*hook_cls, cancel=True)

Clear the given task sets

Parameters

  • hook_cls (str) – task class names

  • cancel (bool) – Cancel the task if a set is not empty, defaults to True. Else will just clear the ref.

async wait(*hook_cls, timeout=None)

Wait for all task in the specific task set(s).

Parameters

timeout (float, optional) – timeout, defaults to None

Return type

Tuple[Set[Task], Set[Task]]

Returns

done, pending

Note

If timeout is None, this method will loop until all tasks in the set(s) are done. That means if other tasks are added during awaiting, the added task will be waited as well.

See also

asyncio.wait()

class aioqzone.interface.hook.LoginEvent

Defines usual events happens during login.

async LoginFailed(msg=None)

Will be emitted on login failed.

Note

This event will be triggered on every failure of login attempt. Means that logic in this event may be executed multiple times.

Parameters

msg (Optional[str]) – Err msg, defaults to None.

async LoginSuccess()

Will be emitted after login success. Low prior, scheduled by loop instead of awaiting.

class aioqzone.interface.hook.QREvent

Defines usual events happens during QR login.

async QrFailed(msg=None)

QR login failed.

Note

This event should always be called before LoginEvent.LoginFailed().

Parameters

msg (Optional[str]) – Error msg, defaults to None.

async QrFetched(png, renew=False)

Will be called on new QR code bytes are fetched. Means this will be triggered on:

  1. QR login start

  2. QR expired

  3. QR is refreshed

Parameters

  • png (bytes) – QR bytes (png format)

  • renew (bool) – this QR is a refreshed QR, defaults to False

Deprecated since version ?: Develpers had better not rely on the parameter renew. Maintaining the state by yourself is not difficult.
